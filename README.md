# PR Review Rulebook

Culture, norms, and governance for how we review pull requests — informed by published practices at Google, Microsoft, GitLab, and Kubernetes, plus the Cisco/SmartBear code review study.

**Live site:** enable GitHub Pages (Settings → Pages → Deploy from branch → `main`, root) and this renders at `https://<user>.github.io/<repo>/` via [index.html](index.html).

Also available as a Word document: [PR Review Rulebook.docx](<PR Review Rulebook.docx>)

## Amending

This rulebook changes by pull request, reviewed like code. Propose a change, the team reviews it, version bumps on merge.
